# Werkstuk

## Stap 1: de database

1. Maak een nieuwe MySQL-database aan met de naam `movies`.
2. Importeer `dump.sql` binnen de database `movies`.

Merk op: in de MySQL-database werden de nodige “comments” ingegeven bij speciale velden. Bekijk deze dus zeker in je SQL-programma!

De volgende site-gebruiker werd reeds aangemaakt in de database:

    e-mailadres: mathias.bynens@arteveldehs.be
    wachtwoord: art3v3ld3

3. Bekijk hoe dit in de database bewaard wordt, en voeg vervolgens via je SQL-programma een account toe voor jezelf. Gebruik hiervoor je Artevelde-e-mailadres, en het wachtwoord `art3v3ld3stud3nt`.

## Stap 2: opzetten van een repository

1. Hernoem de map `CMP2-2MMP-werkstuk-Naam-Voornaam` door `Naam` en `Voornaam` te vervangen door de juiste waarden.
2. Maak een nieuwe private (!) Bitbucket-repository aan met dezelfde naam. Geef mij (Bitbucket-gebruiker `mathiasbynens`) _Admin_-toegang (niet enkel _Read_ of _Write_ dus!).
3. Doe een eerste commit van de bronbestanden in de map `CMP2-2MMP-werkstuk-Naam-Voornaam` naar deze repository, _alvorens_ je enige aanpassingen doet. Synchroniseer deze commit met de online Bitbucket-repository.
4. Doe een tweede commit, waarbij je enkel het bestand `README.md` aanvult met je naam en voornaam. Synchroniseer ook deze commit.
5. Vanaf dat moment moet elke commit in je repository een nieuwe feature of een bugfix van een vorige feature zijn. Vergeet dus niet af en toe te committen! Ik verwacht elke week minstens een paar nieuwe commits te zien.

## Stap 3: PHP, HTML, RSS, Atom en JSON

Bekijk de bestanden in `CMP2-2MMP-werkstuk-Naam-Voornaam`. Elk bestand bevat commentaren; deze commentaren bevatten de nodige instructies.

Hou rekening met de basisprincipes uit de theorielessen! Wie een niet-beveiligde site aflevert, kan onmogelijk slagen.

**Belangrijk:** Wijzig niets aan de mappenstructuur of de bestandsnamen!

## Stap 4: CSS

Voorzie je website van een eigen stijl. Zorg voor een overzichtelijke, verzorgde layout.

## Stap 5: Aanvullen + exporteren database

Voeg minstens 10 films toe aan de database via je eigen gebruikersaccount.

Vervolgens exporteer je de database (zowel de structuur als de data) en bewaar je het resultaat in een bestand genaamd `export.sql` dat je toevoegt aan je repository.
